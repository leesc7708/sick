# 헬스케어 앱 기획서

> **버전 2.0** - 전문가 검토 반영 (앱 개발자, 의사, 약사, 기획자, QA 테스터, 마케터)

## 1. 앱 개요

### 1.1 앱 이름
**"어디아파"**

### 1.2 슬로건
> "증상 확인부터 병원 연결까지, 건강 정보 가이드"

### 1.3 핵심 가치
- **쉬움**: 복잡한 의료 용어 없이 일상 언어로 증상 입력
- **안전함**: Red Flag 시스템으로 응급상황 즉시 안내
- **연결**: 증상 확인에서 끝나지 않고 의료기관까지 연결

### 1.4 서비스 범위 명시
```
⚠️ 본 앱은 의료 참고 정보를 제공하며,
   의료 전문가의 진단이나 처방을 대체하지 않습니다.

   모든 정보는 참고용이며, 정확한 진단과 치료를 위해
   반드시 의료기관을 방문하시기 바랍니다.
```

## 2. 타겟 사용자

### 2.1 주요 타겟
| 타겟 | 특징 | 니즈 |

| **2030 직장인** | 바쁨, 병원 갈 시간 없음 | 빠른 증상 정보, 야간/주말 병원 |

| **노부모 돌봄 자녀** | 부모님 건강 관리 | 원격 건강 확인, 대리 예약 |

### 2.2 사용자 정보 등록 (필수)
```
[기본 정보]
- 연령
- 성별
- 임부/수유부 여부

[건강 정보]
- 기저질환 (당뇨, 고혈압, 간질환, 신장질환 등)
```

### 2.3 사용 시나리오

**시나리오 1: 직장인 김씨 (32세)**
```
토요일 새벽 2시, 갑자기 배가 아픔
→ 앱 실행 → "배 오른쪽 아래가 찌르듯이 아파요"
→ AI 정보 검색: "맹장염 관련 증상 - 의료기관 방문 권장"
→ 근처 응급실 3곳 표시
→ 길찾기 연결
```

**시나리오 2: 엄마 박씨 (38세)**
```
아이가 열이 나고 두드러기
→ 앱 실행 → 증상 체크 + 사진 촬영
→ AI 정보 검색: "알레르기 반응 관련 증상"
→ 긴급도 안내: "내일 소아과 방문 권장"
→ 지금 사용 가능한 해열제 정보 + 반드시 약사 상담 안내
```

## 3. Red Flag 시스템 (필수 기능)

### 3.1 즉시 응급 안내 증상
```
🚨 즉시 119 또는 응급실 안내가 필요한 증상:

1. 흉통 + 호흡곤란
2. 갑자기 발생한 심한 두통 ("인생 최악의 두통")
3. 의식 변화, 혼돈
4. 발열 + 목 경직 (수막염 의심)
5. 심한 복통 + 반발통
6. 안면 마비, 언어 장애, 편측 위약 (뇌졸중 의심)
7. 아나필락시스 증상 (발진 + 호흡곤란 + 저혈압)
8. 심한 출혈
9. 고열 + 의식저하 (영유아)
```

### 3.2 Red Flag 작동 방식
```
[증상 입력 시]
  ↓
[Red Flag 키워드 감지]
  ↓ (해당 시)
┌─────────────────────────────────────┐
│ 🚨 응급 상황 안내                     │
├─────────────────────────────────────┤
│ 입력하신 증상은 응급 상황일 수 있습니다. │
│                                      │
│ 즉시 119에 연락하거나                  │
│ 가까운 응급실을 방문하세요.             │
│                                      │
│ [119 전화 연결]  [응급실 찾기]          │
└─────────────────────────────────────┘

→ 다른 기능으로 진행 불가 (응급 우선)
→ 사용자가 "응급 상황 아님" 확인 시에만 진행
```

## 11. 법적 고려사항

### 11.1 의료법 준수 (Critical)
```
[필수 사항]
- 앱은 "건강 정보 검색 서비스"로 명시
- "진단", "처방", "추천" 용어 사용 금지
- "관련 정보", "안내", "참고용" 용어 사용
- 모든 화면에 면책 고지 표시
- 응급 상황 시 119 연결 최우선 안내

[금지 표현 → 허용 표현]
- "예상 증상" → "관련 가능 상태 (참고용)"
- "추천 진료과" → "관련 진료과 안내"
- "AI 분석" → "AI 기반 정보 검색"
- "진단" → "정보 제공"
```

### 11.2 약사법 준수 (Critical)
```
[필수 사항]
- 약 "추천" 아닌 "정보 제공"으로 명시
- "DUR 연동" → "식약처 공공데이터 기반 상호작용 정보"
- 모든 약 정보에 "약사 상담 필수" 안내
- 처방약 관련 조언 금지

[금지 표현 → 허용 표현]
- "일반의약품 추천" → "일반의약품 정보 제공 + 약사 상담 권장"
- "복용하세요" → "약사와 상담하세요"
```

### 11.3 개인정보보호
- 건강정보 = 민감정보, 별도 동의 필요
- 데이터 암호화 저장 (AES-256)
- 개인정보처리방침 명시
- ISMS-P 인증 추진
