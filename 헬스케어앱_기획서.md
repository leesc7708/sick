# 헬스케어 앱 기획서

## 1. 앱 개요

### 1.1 앱 이름 (가칭)
**"어디아파"** 또는 **"닥터가이드"**

### 1.2 슬로건
> "증상부터 처방까지, 원스톱 건강 가이드"

### 1.3 핵심 가치
- **쉬움**: 복잡한 의료 용어 없이 일상 언어로 증상 입력
- **정확함**: 한국 의료 환경에 최적화된 진료과/약품 추천
- **연결**: 증상 확인에서 끝나지 않고 실제 진료까지 연결

## 2. 타겟 사용자

### 2.1 주요 타겟
| 타겟 | 특징 | 니즈 |
|------|------|------|
| **2030 직장인** | 바쁨, 병원 갈 시간 없음 | 빠른 증상 체크, 야간/주말 병원 |
| **부모 (3040)** | 아이 아플 때 당황 | 소아과? 응급실? 빠른 판단 |
| **5060 중장년** | 만성질환, 다약제 복용 | 약 상호작용 체크, 복약 관리 |
| **노부모 돌봄 자녀** | 부모님 건강 관리 | 원격 건강 확인, 대리 예약 |

### 2.2 사용 시나리오

**시나리오 1: 직장인 김씨 (32세)**
```
토요일 새벽 2시, 갑자기 배가 아픔
→ 앱 실행 → "배 오른쪽 아래가 찌르듯이 아파요"
→ AI 분석: "맹장염 의심, 응급실 방문 권장"
→ 근처 응급실 3곳 + 현재 대기시간 표시
→ 길찾기 연결
```

**시나리오 2: 엄마 박씨 (38세)**
```
아이가 열이 나고 두드러기
→ 앱 실행 → 증상 체크 + 사진 촬영
→ AI 분석: "알레르기 반응 의심"
→ 응급 여부: "긴급하지 않음, 내일 소아과 방문"
→ 지금 먹일 수 있는 해열제 추천 + 주의사항
```

**시나리오 3: 어르신 최씨 (67세)**
```
고혈압약 + 당뇨약 복용 중, 감기약 사려고 함
→ 앱에서 현재 복용약 등록
→ 약국에서 감기약 바코드 스캔
→ "이 감기약은 혈압약과 상호작용 있음"
→ 대체 약품 3개 추천
```

## 3. 핵심 기능 (MVP)

```
2. 신체 부위 선택: 3D 인체 모형에서 터치
3. 음성 입력: 말로 설명
4. 사진 첨부: 피부 증상 등

[AI 분석]
```

## 4. 기술 구조

### 4.1 시스템 아키텍처
```
┌─────────────────────────────────────────────┐
│                  Frontend                    │
│         (React Native / Flutter)             │

└─────────────────┬───────────────────────────┘
                  │ API
┌─────────────────▼───────────────────────────┐
│                  Backend                     │
│            (Node.js / Python)                │
├─────────────────────────────────────────────┤
│  - 사용자 인증      - 증상 분석 API          │
│  - 약품 DB 관리     - 병원 검색 API          │
│  - 알림 서비스      - AI 엔진 연동           │
└─────────────────┬───────────────────────────┘
                  │
┌─────────────────▼───────────────────────────┐
│              External APIs                   │
├─────────────────────────────────────────────┤
│  - 식약처 의약품 DB (공공데이터)             │

│  - 네이버/카카오 지도 API                    │

└─────────────────────────────────────────────┘
```

### 4.2 핵심 데이터베이스

#### 의약품 DB
```
- 식약처 의약품안전나라 API
- 약 65,000개 품목
- DUR (의약품안전사용서비스) 정보
- 병용금기, 연령금기, 임부금기 등
```

#### 증상-질환 DB
```
- 증상 키워드: 약 2,000개
- 질환 정보: 약 5,000개
- 증상-질환 매핑
- 진료과 매핑
```

#### 병원/약국 DB
```
- 건강보험심사평가원 API
- 전국 병원 약 70,000개
- 전국 약국 약 24,000개
- 진료과목, 운영시간, 위치
```

### 4.3 AI 모델

#### 증상 분석 엔진
```
[Input]
"3일 전부터 기침이 나고 열이 38도예요.
 콧물도 나고 목이 아파요."

[Processing]
1. 자연어 처리 (NLP)
   - 증상 추출: 기침, 발열(38도), 콧물, 인후통
   - 기간 추출: 3일

2. 증상 매칭
   - 감기 (85%)
   - 독감 (72%)
   - 코로나19 (45%)
   - 편도염 (38%)

3. 긴급도 판단
   - 고열 지속 → 진료 권장

[Output]
- 예상 질환 + 확률
- 추천 진료과
- 긴급도 레벨
- 셀프케어 팁
```

## 9. 법적 고려사항

### 9.1 의료법 준수
- 앱은 "의료 참고 정보" 제공, 진단/처방 아님 명시
- "이 정보는 의료 전문가의 조언을 대체하지 않습니다" 고지
- 응급 상황 시 119 연결 안내

### 9.2 개인정보보호
- 건강정보 = 민감정보, 별도 동의 필요
- 데이터 암호화 저장
- 개인정보처리방침 명시

### 9.3 의약품 정보
- 식약처 공공데이터 활용 (출처 명시)
- 약 추천 시 "약사와 상담하세요" 안내
