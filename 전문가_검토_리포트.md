## 🔴 즉시 수정 필요 (Critical)

### 1. 법적 리스크 해결

| 현재 | 수정 필요 |
|------|----------|
| 슬로건: "증상부터 **처방**까지" | "증상 확인부터 **병원 연결**까지" |
| "예상 증상: 편두통 (78%)" | "관련 가능 상태: 편두통 (참고용)" |
| "추천 진료과" | "관련 진료과 안내" |
| 일반의약품 "추천" | 일반의약품 "정보 제공" + "약사 상담 필수" |
| "AI 분석" | "AI 기반 정보 검색" |
| "DUR 연동" | "식약처 공공데이터 기반 상호작용 정보" |

### 2. Red Flag(위험 신호) 시스템 필수 도입

```
[즉시 응급실/119 안내 필요 증상]
- 흉통 + 호흡곤란
- 갑자기 발생한 심한 두통 ("인생 최악의 두통")
- 의식 변화, 혼돈
- 발열 + 목 경직
- 심한 복통 + 반발통
- 안면 마비, 언어 장애, 편측 위약 (뇌졸중 의심)
- 아나필락시스 증상 (발진 + 호흡곤란 + 저혈압)
```

## 🟡 수정 권장 (Important)

### 개발 관련
- MVP 기간: 3개월 → **5~6개월**
- 베타 테스터: 100명 → **500~1,000명**
- 앱 개발자: 2명 → **3명** (iOS, Android, 공통)
- QA 인력 **1명 추가** 필요
- 실시간 대기 현황 기능: **Phase 2로 연기 또는 제외**

### 수익 모델
- 프리미엄 전환율: 3% → **1.5%** (보수적 추정)
- 연 매출 목표: 12억원 → **6~8억원**

### 기능 관련
- 약 위험도: 3단계 → **5단계**로 세분화
- 약국 재고 표시: **Phase 3 이후로 연기**

---

## ✅ 추가 필요 기능

### 필수 추가
1. **의료진 자문단** 구성 (내과, 응급의학과, 소아과, 가정의학과 최소 4~5명)
2. **약사 자문단** 구성 (콘텐츠 검수)
3. **온보딩 플로우** 설계
4. **오프라인 대응** (복약 알림 등 핵심 기능)
5. **접근성** (WCAG 2.1 AA, 고령자/장애인 대응)

### 마케팅 추가
1. **추천인 리워드** 프로그램
2. **공유 기능** (증상 체크 결과 카드)
3. **ASO 전략** (키워드, 스크린샷)
4. **런칭 캠페인** (소프트/하드 런칭)

### 사용자 정보 추가
1. 연령, 성별, 임부/수유부 여부 등록
2. 기저질환 등록 (맞춤 긴급도 조정)
3. 연령별 용량 정보

## 👍 잘된 점

1. **명확한 타겟 정의**: 4개 페르소나와 사용 시나리오가 구체적
2. **단계적 MVP 전략**: Phase별 기능 분리가 현실적
3. **차별화 포인트**: 한국 의료 환경 특화, 원스톱 서비스
4. **공공 API 활용 계획**: 식약처, 심평원 데이터 활용 현실적
5. **복약 순응도 관리**: 만성질환 관리에 실질적 가치
6. **법적 리스크 인지**: 의료법, 개인정보보호 이슈 사전 파악

## 🎯 결론

기획서의 **방향성과 기능 설계는 우수**하나, 다음 3가지 핵심 이슈 해결이 필수입니다:

1. **의료법/약사법 저촉 위험** → 표현 전면 수정 + 자문단 구성
2. **마케팅 전략 부재** → 예산 확보 + 채널/런칭 전략 수립
3. **개발 일정 과소평가** → 현실적 일정으로 재조정

이 3가지가 해결되면, 국내 헬스케어 앱 시장에서 **경쟁력 있는 서비스**로 성장할 수 있습니다.
