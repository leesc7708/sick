# 어디아파 (WhereHurts)

> 증상 확인부터 병원 연결까지, 건강 정보 가이드 앱

## 프로젝트 개요

일반인이 증상을 느꼈을 때 어떤 진료과를 가야 할지, 어떤 약을 먹어도 되는지 쉽게 알 수 있도록 건강 정보를 제공하는 헬스케어 앱입니다.

**주의**: 본 앱은 의료 참고 정보를 제공하며, 의료 전문가의 진단이나 처방을 대체하지 않습니다.

## 핵심 기능

### 1. 증상 정보 검색
- 자연어/음성으로 증상 입력
- AI 기반 정보 검색
- 관련 진료과 안내
- **Red Flag 시스템**: 응급 증상 즉시 119 안내

### 2. 병원 찾기
- 관련 진료과 기반 병원 검색
- 예상 진료비 안내 (심평원 기준)
- 예약 연동

### 3. 약 상호작용 정보
- 복용 중인 약 등록
- 약-약, 약-음식 상호작용 정보 (식약처 공공데이터 기반)
- 일반의약품 정보 제공 (약사 상담 필수 안내)

### 4. 복약 알림
- 시간별 복약 알림
- 복약 준수율 통계
- 가족 건강 관리

## 전문가 검토 요약

| 전문가 | 주요 의견 | 반영 상태 |
|--------|----------|----------|
| 앱 개발자 | 개발 기간/비용 재조정 | ✅ 반영 |
| 의사 | 오진단 위험, 의료법 | ✅ 반영 |
| 약사 | 약사법, 표현 수정 | ✅ 반영 |
| 기획자 | 리텐션 전략, 수익모델 | ✅ 반영 |
| QA 테스터 | 접근성 가이드라인 | ✅ 반영 |
| 마케터 | 마케팅 전략 추가 | ✅ 반영 |

## v2.0 주요 변경사항

- Red Flag 시스템 추가 (응급 증상 즉시 안내)
- 법적 표현 전면 수정 ("추천" → "정보 제공")
- 마케팅 전략 및 예산 추가 (3억원)
- 개발 일정 현실화 (3개월 → 5~6개월)
- 의료진/약사 자문단 필수 항목 추가
- 접근성 가이드라인 (WCAG 2.1 AA)
- 총 예산: 7~7.5억원

## 기술 스택 (예정)

- **Frontend**: React Native / Flutter
- **Backend**: Node.js / Python
- **AI**: OpenAI API / Claude API
- **Database**: PostgreSQL, Redis
- **APIs**: 식약처 공공데이터, 건강보험심사평가원, 네이버 지도
